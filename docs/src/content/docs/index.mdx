## Quick Example

```typescript
import { createSQLiteClient } from "@alexop/sqlite-core";
import { z } from "zod";

// Define schema with Zod
const schema = {
  todos: z.object({
    id: z.string(),
    title: z.string(),
    completed: z.boolean().default(false),
    createdAt: z.string().default(() => new Date().toISOString()),
  }),
} as const;

// Create client
const db = await createSQLiteClient({
  schema,
  filename: "file:app.sqlite3?vfs=opfs",
  migrations: [
    {
      version: 1,
      sql: `CREATE TABLE todos (
      id TEXT PRIMARY KEY,
      title TEXT NOT NULL,
      completed INTEGER DEFAULT 0,
      createdAt TEXT DEFAULT CURRENT_TIMESTAMP
    )`,
    },
  ],
});

// Query with full type safety
const todos = await db
  .query("todos")
  .where("completed", "=", false)
  .orderBy("createdAt", "DESC")
  .all();

// Insert with validation
await db.insert("todos").values({
  id: crypto.randomUUID(),
  title: "Build something awesome",
});
```

## Architecture

SQLite Web is a monorepo with two main packages:

- **`@alexop/sqlite-core`** - Framework-agnostic SQLite client using sqlite-wasm
- **`@alexop/sqlite-vue`** - Vue 3 plugin and composables for reactive queries

Both packages are built with TypeScript and provide full type safety throughout your application.

## Browser Requirements

SQLite WASM requires specific headers for `SharedArrayBuffer` support:

```typescript
// vite.config.ts
export default {
  server: {
    headers: {
      "Cross-Origin-Opener-Policy": "same-origin",
      "Cross-Origin-Embedder-Policy": "require-corp",
    },
  },
};
```

See the [Browser Setup](/guides/browser-setup/) guide for complete configuration details.
